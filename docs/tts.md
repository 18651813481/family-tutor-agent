# TTS 语音家教线程

本项目把 TTS 视为核心能力。学生辅导线程应该能把老师回复读出来，帮助孩子跟读、听懂和保持参与。

本文件只定义抽象接口和行为要求，不绑定具体 TTS 实现。

## 线程边界

- 学生辅导线程：允许发声。
- 家长管理线程：默认静音。
- 系统维护、代码执行、日志输出、调试信息：默认不读。

推荐使用线程 allowlist 或等价机制，只允许指定学生线程进入 TTS。

## TTS 输入

TTS 应读取 assistant 的最终自然语言回复。

应该朗读：

- 中文讲解。
- 英文单词、短语和句子。
- 普通项目符号中的学习内容。
- 老师带读、逐句讲解和小练习。

不应该朗读：

- 代码块。
- shell 命令。
- 文件路径。
- URL。
- JSON。
- 日志。
- 密钥、token、邮箱或配置项。
- 数据库查询结果。

## 英文不能跳读

自然语言里的英文必须保留给 TTS。

正确：

- First Aid Center
- What did you do yesterday?
- I swam yesterday.
- 这个单词 swim 是“游泳”。

错误：

- 只读中文解释，跳过 `swim`。
- 逐句讲解时不读英文原句。
- 把英文和中文混在同一行，导致孩子只听中文。

## 英语教学输出规则

学英语时，推荐格式是：

- 先给英文。
- 让孩子读。
- 再解释中文。
- 最后给一个小练习。

示例：

先读这一句：

I swam yesterday.

你先读一遍，再猜猜 `swam` 是现在做，还是昨天做？

## 过滤规则建议

TTS 过滤器应该保留自然语言英文，同时过滤技术文本。

保留条件：

- 包含英文单词或中文学习内容。
- 看起来像自然语言句子、词组、例句或讲解。

过滤条件：

- 看起来像命令、路径、配置、日志、JSON、长错误堆栈或密钥。
- 包含大量符号、斜杠、等号、花括号或不可读机器文本。

## 晚间听力建议

如果孩子晚上正在学生线程学习，且接近收尾，可以轻轻建议听 5-10 分钟英语音频。不要创建固定提醒，也不要打断正在做的题。

示例：

今晚如果还想轻松练一下英语，可以选一个单元音频听 5 分钟。你想听第几单元？
